## Overview

The Procta API is organized around REST. All endpoints accept and return JSON, use standard HTTP methods, and return conventional HTTP status codes.

## Base URL

```
https://api.procta.org/v1
```

## Authentication

Most endpoints require a Bearer token in the `Authorization` header. See [Authentication](/authentication) for details on key types and management.

```
Authorization: Bearer prc_owner_abc123...
```

## API Categories

| Category | Description |
| --- | --- |
| **Health** | Service health and readiness checks |
| **Auth** | Login, identity resolution, password management |
| **Owners** | Owner registration, KYC, and account management |
| **Relying Parties** | Relying party registration and configuration |
| **Agents** | Agent CRUD, lifecycle (suspend/revoke/reinstate), key rotation |
| **Credentials** | Verifiable credential issuance, verification, and reissuance |
| **Verification** | Real-time agent action verification |
| **Scopes** | Predefined scope registry and lookup |
| **Audit** | Audit trail queries with pagination and CSV export |
| **Webhooks** | External service webhook handlers (Sumsub KYC) |
